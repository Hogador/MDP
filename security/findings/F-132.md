### F-132 [HIGH] GuardianUserOpBuilder не использует paymaster (veto не бесплатный)

| Field | Value |
|-------|-------|
| **ID** | F-132 |
| **Severity** | HIGH |
| **Category** | C6 (paymaster) |
| **Bug pattern** | `guardian-no-paymaster` |
| **Fingerprint** | `sha256("C6:app:GuardianUserOpBuilder:guardian-no-paymaster")` → `f132...` |
| **Status** | NEW |
| **Found** | Architecture Audit 2026-06-30 |

**Description:**
`GuardianUserOpBuilder.buildUserOp()` хардкодит `paymasterAndData = ByteArray(0)` для confirmGuardian, approveRecovery, vetoRecovery. PRD §7: "Veto = бесплатный (Paymaster покрывает gas)".

Guardian без BNB не может veto → атакующий доводит recovery до execute.

Класс уже инжектит `PaymasterClient`, но не использует его. Блокируется F-130 (API mismatch — PaymasterClient не может корректно запросить подпись).

**Impact:** HIGH — veto недоступен для guardian'ов без BNB.

**Required fix:**
1. Исправить F-130 (PaymasterClient API)
2. Переиспользовать `PaymasterClient.signUserOp()` в `GuardianUserOpBuilder.buildUserOp()`
3. Проверить whitelist MDAOPaymaster для SocialRecoveryModule callData
