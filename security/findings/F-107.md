### F-107 [HIGH] TrustProviderRegistry без timelock

| Field | Value |
|-------|-------|
| **ID** | F-107 |
| **Severity** | HIGH |
| **Category** | C1 (access-control) |
| **Bug pattern** | `registry-no-timelock` |
| **Fingerprint** | `sha256("C1:contracts/TrustProviderRegistry.sol:registerProvider:registry-no-timelock")` → `a107000000000000000000000000000000000000000000000000000000000001` |
| **Status** | NEW |
| **Found** | External Cloud Audit 2026-06-29 |

**Description:**
TrustProviderRegistry.registerProvider/setProviderStatus — onlyOwner без timelock. MDAOPaymaster.setRegistry() защищён TimelockController, но registry owner может быть EOA.

**Impact:**
HIGH — компрометация одного EOA-ключа меняет верификатора подписи в обход timelock.

**Verification recipe:**
```bash
grep "transferOwnership.*timelock\|owner.*timelock" contracts/script/Deploy.s.sol
# PASS: registry.transferOwnership(address(timelock))
```

**Required fix:**
В Deploy.s.sol: registry.transferOwnership(address(timelock)). proposers = [gnosisSafe], admin = address(0).
