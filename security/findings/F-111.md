### F-111 [HIGH] ALLOW_LOCAL_SIGNING production guard отсутствует

| Field | Value |
|-------|-------|
| **ID** | F-111 |
| **Severity** | HIGH |
| **Category** | D2 (config) |
| **Bug pattern** | `local-signing-production` |
| **Fingerprint** | `sha256("D2:backend:AppConfig.kt:local-signing-production")` → `f111...` |
| **Status** | CLAIMED_FIXED |
| **Found** | BSC Testnet Audit 2026-06-29 |
| **Fixed** | 2026-07-01 |

**Description:** Нет `isProduction()` guard. Production может запуститься с ALLOW_LOCAL_SIGNING=true и приватным ключом в env. init-block guard обходится try-catch.

**Impact:** HIGH — компрометация paymaster ключа.

**Fix:** Runtime guard в getter (AppConfig.kt:43-49). `allowLocalSigning` — body property с `get()`:
```kotlin
var allowLocalSigning: Boolean = false
    get() {
        if (!isTestnet && field) {
            throw IllegalStateException("ALLOW_LOCAL_SIGNING is forbidden in production. Use KMS or HSM.")
        }
        return field
    }
```
Guard срабатывает при доступе к свойству, а не при конструировании (init-block обходится try-catch). `fromEnv()` устанавливает значение через setter после конструирования.

**Verification:**
```bash
./gradlew test --tests "com.mdaopay.paymaster.AppConfigValidationTest"
# F-111 rejects ALLOW_LOCAL_SIGNING on production — runtime guard throws on access
# F-111 allows ALLOW_LOCAL_SIGNING on testnet
```
