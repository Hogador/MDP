### F-100 [CRITICAL] Paymaster не используется в send-флоу

| Field | Value |
|-------|-------|
| **ID** | F-100 |
| **Severity** | CRITICAL |
| **Category** | G1 (architecture) |
| **Bug pattern** | `paymaster-unused-send` |
| **Fingerprint** | `sha256("G1:app/.../SendRepository.kt:sendUsdt:paymaster-unused-send")` → `a100000000000000000000000000000000000000000000000000000000000001` |
| **Status** | CLAIMED_FIXED |
| **Found** | External Cloud Audit 2026-06-29 |

**Description:**
SendRepository.sendUsdt() строит UserOp с paymasterAndData = ByteArray(0). PaymasterClient.encodePaymasterAndData() существует, но не вызывается. GaslessTransactionOrchestrator отсутствует.

**Impact:**
CRITICAL — PRD-обещание gasless UX не работает. Пользователи платят BNB за газ.

**Verification recipe:**
```bash
grep "paymasterClient.getQuote" app/src/main/java/com/mdaopay/app/domain/usecase/GaslessTransactionOrchestrator.kt
# PASS: getQuote вызывается в sendUsdtGasless
grep "paymasterAndData = quote" app/src/main/java/com/mdaopay/app/domain/usecase/GaslessTransactionOrchestrator.kt
# PASS: paymasterAndData из quote передаётся в buildUserOp
```

**Required fix:**
Создать GaslessTransactionOrchestrator между SendViewModel и SendRepository:
1. Запросить /sign quote через PaymasterClient
2. Подставить paymasterAndData в UserOp
3. При ошибке quote — fallback на native-gas с UI уведомлением
