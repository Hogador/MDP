### F-101 [CRITICAL] Guardian on-chain флоу отсутствует в клиенте

| Field | Value |
|-------|-------|
| **ID** | F-101 |
| **Severity** | CRITICAL |
| **Category** | G1 (architecture) |
| **Bug pattern** | `guardian-onchain-missing` |
| **Fingerprint** | `sha256("G1:app/.../GuardianManager.kt:acceptInvite:guardian-onchain-missing")` → `a101000000000000000000000000000000000000000000000000000000000001` |

**Description:**
GuardianManager оперирует исключительно через RelayClient. on-chain методы SocialRecoveryModule.sol (addGuardian, confirmGuardian, approveRecovery, vetoRecovery) не вызываются. GuardianInfo.pubKeyX/pubKeyY пустые.

**Impact:**
CRITICAL — основная фича (Trusted Contacts Recovery) не функциональна end-to-end. Блокер для testnet.

**Verification recipe:**
```bash
grep -r "confirmGuardian\|approveRecovery\|vetoRecovery" app/**/*.kt
# PASS: есть вызовы on-chain методов
```

**Required fix:**
Создать GuardianUserOpBuilder:
1. Генерация P-256 ключа через PasskeyManager при acceptInvite
2. UserOp на confirmGuardian с реальными pubKeyX/pubKeyY
3. approveRecovery/vetoRecovery через WebAuthn assertion с real signatures
