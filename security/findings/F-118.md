### F-118 [MEDIUM] SessionKeyModule — нет on-chain selector whitelist

| Field | Value |

| **Severity** | MEDIUM |
| **Category** | C3 (access) |

| **Fingerprint** | `sha256("C3:SessionKeyModule.sol:permissions:selector-whitelist-missing")` → `f118...` |
| **Status** | CLAIMED_FIXED |

| **Fixed** | 2026-07-01 (Wave 2) |

**Description:** Любой permission может быть добавлен. Client-side validation обходится прямой отправкой UserOp.

**Required fix:** `mapping(bytes32 => bool) allowedPermissions` + owner-only add + validation в createSessionKey.

**Fix applied:**
```solidity
// SessionKeyModule.sol
address public owner;
mapping(bytes32 => bool) public allowedPermissions;
uint256 public allowedPermissionCount;

modifier onlyOwner() { if (msg.sender != owner) revert Unauthorized(); _; }
constructor() { owner = msg.sender; }

function setPermissionAllowed(bytes32 permission, bool allowed) external onlyOwner { ... }

// createSessionKey:
if (allowedPermissionCount > 0) {
    for (uint256 i = 0; i < permissions.length; i++) {
        if (!allowedPermissions[permissions[i]]) revert PermissionNotAllowed();
    }
}
```
Когда whitelist пуст (allowedPermissionCount == 0), все permissions разрешены — backwards compatible.

**Verification:**
```bash
forge test --match-contract SessionKeyModule -vvv
# PASS: PermissionNotAllowed при неразрешённом permission
# PASS: createSessionKey работает без whitelist (allowedPermissionCount == 0)
```
