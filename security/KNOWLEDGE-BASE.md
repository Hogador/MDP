# KNOWLEDGE BASE — MDAOPay

> Реестр верифицированных правил. Каждое правило прошло проверку
> через verifier --logic или verifier --requirements и подтверждено.
>
> Никаких правил без верификации. Никаких "вроде так лучше".
>
> Формат ID правила: KB-<DOMAIN>-<NNN> где DOMAIN in
> {SOL, MOB, BACK, INF, SEC, UX}.

---

## Формат правила

~~~markdown
### KB-XXX-NNN: <короткое название>

**Утверждение:** <что утверждается>

**Верификация:** verifier --logic | verifier --requirements
**Источник:** <задача / ADR / audit finding>
**Дата:** YYYY-MM-DD

**Доказательство:**
<код / тест / ссылка на ADR>

**Контр-пример (если применимо):**
<что произойдёт без этого правила>
~~~
