## 1. Что такое MDAOPay

MDAOPay — это DAO-управляемый платёжный протокол поверх EVM-совместимых
сетей, с mobile-first кошельком для конечного пользователя. Протокол
обеспечивает:

- Кастодиальное хранение средств (приватные ключи только на устройстве)
- P2P-переводы между участниками DAO
- Раздельные платежи (payment splitting) по предложениям DAO
- Восстановление доступа без контроля над приватным ключом (social recovery)
- Голосование по казне DAO непосредственно из кошелька

## 2. Что МЫ ДЕЛАЕМ (in scope)

| Домен | Делаем |
|---|---|
| Кошелёк | Mobile (Android/iOS), EVM-only, BLS-подписи |
| Смартконтракты | Solidity ^0.8.x, OpenZeppelin base, минимальная внешняя поверхность |
| Сеть | Любая EVM-совместимая L1/L2 (config-driven) |
| Управление | On-chain DAO (голосование = tx, не off-chain) |
| Восстановление | Social recovery через guardians (BLS-based) |
| Деплой | Reproducible builds, мульти-подпись для admin-функций |

---

## 3. Что МЫ НЕ ДЕЛАЕМ (out of scope)

| Не делаем | Почему |
|---|---|
| Custodial-решения | Противоречит базовому принципу self-custody |
| KYC/AML | Не наша ответственность — это работа фиатовых шлюзов |
| Собственный L1/L2 | Используем существующие сети |
| Non-EVM (Solana, Bitcoin) | Узкая область — лучше сделать одно хорошо |
| Фиатный шлюз | Out of scope — интегрируем сторонние, не строим свой |
| DEX/AMM | Это отдельный продукт, не платёжный протокол |
| NFT-маркетплейс | Не связано с основной миссией |
| Web-only кошелёк | Mobile-first; web может быть view-only |

## 4. ЗАПРЕЩЁННЫЕ КОМПРОМИССЫ

Эти принципы не могут быть нарушены ни при каких обстоятельствах —
ни ради скорости, ни ради совместимости, ни ради "временного решения".

### 4.1. Безопасность ключей
- Запрещено: приватный ключ покидает устройство пользователя
- Запрещено: hot-wallet с admin-ключами в продакшене
- Запрещено: экспорт приватного ключа в plaintext

### 4.2. Смартконтракты
- Запрещено: OWNABLE upgradeable patterns без Timelock >= 48ч
- Запрещено: проверки через tx.origin
- Запрещено: inline assembly для криптографии (только audited libs)
- Запрещено: delegatecall к контрактам не из whitelisted набора

### 4.3. Управление
- Запрещено: admin-функции без multisig (>= 3 из 5)
- Запрещено: emergency-pause без on-chain голосования для разблокировки
- Запрещено: mint-функции у governance-токена после деплоя

### 4.4. Приватность
- Запрещено: телеметрия по транзакциям пользователя
- Запрещено: analytics SDK в мобильном приложении без opt-in

### 4.5. Деплой
- Запрещено: приватники в CI/CD (только HSM или KMS)
- Запрещено: forge script без --verify на mainnet
- Запрещено: непроверенные контракты на mainnet
