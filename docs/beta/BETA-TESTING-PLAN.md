# Beta Testing Plan

## Phase 1: Closed Alpha (Week 1-2)
- 10 testers (MarsDAO team)
- Focus: Core flows (send, receive, recovery)
- Feedback: Daily standup + bug reports

## Phase 2: Closed Beta (Week 3-4)
- 50 testers (MarsDAO community)
- Focus: Full feature set + edge cases
- Feedback: GitHub issues + weekly sync

## Phase 3: Open Beta (Week 5-6)
- 100+ users (public)
- Focus: Scale, performance, UX polish
- Feedback: In-app feedback + analytics

## Test Scenarios
1. Onboarding → Send → Receive (happy path)
2. Recovery flow (guardian approval)
3. Session Key connect (Identity Connect)
4. Error handling (network, insufficient funds)
5. Theme switching (dark/light/amoled)

## Success Criteria
- < 1% crash rate
- < 5% error rate on /sign
- p95 latency < 2s
- 80%+ user satisfaction
